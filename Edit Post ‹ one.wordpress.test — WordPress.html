Comparing Vagrant and Docker

Sarah Photowat

July 12, 2020

I have not used Vagrant in the past, so I had to do some research.

Docker features are:  it is open source, it started in 2013, it is a Linux-only virtual environment (VE) tool and not a VM tool, it builds on Linux Containers that enable creation and running of multiple Linux VE on a single host, it does not create its own virtual computer with a distinct OS and processors and hardware emulation, it sits on existing kernel’s image of the underlying hardware, and only creates a ‘container’ and it places only a little extra load on the system, it has a very little overhead because of the shared kernel, its isolation is not as good as a full VM’s.

Vagrant features are: It is an open-source, the product released in 2010, it is best described as a VM manager, it allows user to script and package the VM config and the provisioning of the setup, it is designed to run on top of almost any VM tool – VirtualBox, VMWare, AWS, etc. However, default support is only included for VirtualBox.

Differences are:

  • Docker is a Virtual Environment (VE), Vagrant is a Virtual Machine (VM)
  • Docker has little overhead due to used share kernel, whereas Vagrant has more overhead using its own OS
  • Docker relies on the host OS; Vagrant includes the OS within itself
  • Docker containers only run on Linux. Vagrant files can contain any OS and it needs to within Linux VM

Which one do I prefer; Docker or Vagrant? and WHY?

I prefer Vagrant as it helped me do my homework-7 much faster than previous homework. For homework-7 I did not need to install OS in virtual box, configure the machine, investigate on how to install WordPress and configure WordPress.  Using Vagrant felt like a tool that helped me build and manage virtual machine environments in a workflow with couple of easy steps. With this easy-to-use workflow, I was able to focus rather on content of my WordPress page versus installation and configuration of OS and WordPress application. I felt I am using an automation tool and reusing an image that someone built for me before. I believe Vagrant lowered my effort to setup development environment and saved me time. It made my job easier as an acting web-site developer.

I did some reading and found out that Vagrant and Docker are not necessarily competitor and sometimes they can complement each other. Some admins use Vagrant to create a base VM, then when they need to create different configs that all utilize this base VM, use Docker to provision and create different lightweight versions.

References

“Vagrant versus Docker”, Sumo logic, https://www.sumologic.com/blog/vagrant-vs-docker

“Docekr v. Vagrant” Upguard,  https://www.upguard.com/blog/docker-vs-vagrant
